Ресурс Instance

Instance — представляет собой введенную абстракцию, созданную для обеспечения изолированного вычислительного окружения, предназначенного для развертывания и выполнения приложений или сервисов. Основная цель instance — предоставить клиенту управляемый экземпляр ресурсов, где можно настраивать конфигурацию рабочих нагрузок, контролировать использование ресурсов и квоты, отслеживать состояние запущенных процессов и интегрироваться с инструментами DevOps для эффективного управления и масштабирования приложений.

Поля ресурса​

  • Описание
  • Структура
  • Ограничения
  • Валидация
  • spec.spi - доступность персональной информации.
  • spec.chatid - идентификатор чата в telegram для отпроавки уведомлений.
  • spec.description - краткое описание инстанса.
  • spec.quotas - квоты на ресурсы проекта, разделенные на лимиты и запросы для CPU, RAM и DISK.
  • spec.quotas.limits - лимиты для CPU, RAM в проекте.
  • spec.quotas.limits.cpu - лимит по CPU.
  • spec.quotas.limits.memory - лимит по RAM.
  • spec.quotas.requests - запросы для CPU, RAM в проекте.
  • spec.quotas.requests.cpu - запросы на CPU.
  • spec.quotas.requests.memory - запросы на RAM.
  • spec.quotas.requests.storage - запросы на DISK.
  • status - структура, определяющая текущее состояние проекта.
  • status.conditions[] - массив, содержащий условия текущего состояния проекта.
  • status.conditions[].lastTransitionTime - время последнего изменения состояния.
  • status.conditions[].message - сообщение с деталями о переходе состояния.
  • status.conditions[].observedGeneration - наблюдаемая версия объекта.
  • status.conditions[].reason - идентификатор причины перехода состояния.
  • status.conditions[].status - итоговое состояние.
  • status.conditions[].type - тип условия.
  • status.status - текущее состояние проекта.

  • Валидация создания ресурса

    • metadata.labels:

      • Ключи и значения меток должны быть строками.
      • Максимальная длина ключей и значений: 63 символа.
      • Допускаются только буквы, цифры, дефисы, точки и символ _.
      • Не допускается дублирование ключей.

    • metadata.name:

      • Имя должно быть строкой.
      • Максимальная длина имени: 253 символа.
      • Допускаются только буквы, цифры (0-9), дефисы (-), и символ подчеркивания (_).
      • Имя не должно начинаться или заканчиваться дефисом (-) или подчеркиванием (_).
      • Полное имя экземпляра (сочетание имени и префикса) проверяется на уникальность и корректность.

    • spec.description:

      • Описание должно быть строкой.
      • Максимальная длина описания: 255 символа.
      • Могут использоваться любые символы, поддерживаемые UTF-8.

    • spec.chatID:

      • Значение идентификатора чата должно быть строкой.
      • Допускаются только буквы, цифры, дефисы (-), и точки (.).
      • Максимальная длина chatID: 59 символов.

    • spec.quotas:

      • Лимиты квот должны быть положительными числами.
      • Каждый лимит не должен превышать hard limits, установленные на уровне проекта.
      • Отрицательные значения квот не допускаются.
      • При вычислении ресурсов должны учитываться квоты всех инстансов в текущем проекте, за исключением текущего инстанса.
      • Проверки производятся на основе агрегации ресурсов инстанса и проекта.
