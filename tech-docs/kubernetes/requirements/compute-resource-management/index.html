2.6.1. Требования к системе управления вычислительными ресурсами​

№Требование
3Платформа должна поддерживать возможность создания виртуальных машин на основе предустановленных шаблонов или ISO-образов операционных систем.int +out +
5Платформа должна обеспечивать возможность создания виртуальных машин с графическим адаптером.int +out +
6Платформа должна обеспечивать возможность подключения bare-metal графических адаптеров.int +out +
7Платформа должна обеспечивать возможность конфигурирования платформы виртуализации для проброса графического адаптера целиком в ВМ.int +out +
8Платформа должна обеспечивать возможность конфигурирования платформы виртуализации для проброса части графического адаптера в ВМ.int +out +
9Платформа должна обеспечивать поддержку управления ВМ с гостевыми операционными системами Linux.int +out +
10Платформа должна обеспечивать возможность создания отказоустойчивых кластеров для платформы контейнеризации, состоящих не менее, чем из 3 узлов.int +out +
11Платформа должна обеспечивать возможность создания отказоустойчивых кластеров для СУБД на Linux, состоящих не менее, чем из 3 узлов с возможностью выбора необходимой конфигурации по каталогу.int +out +
12Узел выполнения нагрузки может быть добавлен или удален из экземпляра платформы (кластера) стандартными методами, узел выполнения нагрузки может выполнять и виртуализированную нагрузку, в частности ВМ.
13Платформа должна поддерживать добавление vCPU виртуальных машин без прерывания их работы.
14Платформа должна поддерживать расширение RAM виртуальных машин без прерывания их работы.
15Платформа должна поддерживать добавление, расширение дисков виртуальных машин без прерывания их работы.
16Платформа должна поддерживать изменение параметров сетевых адаптеров виртуальных машин без прерывания их работы.
17Платформа должна поддерживать добавление виртуальных сетевых адаптеров виртуальных машин без прерывания их работы.
18Платформа должна поддерживать функции высокой доступности для PaaS/SaaS (High Availability).
19Платформа должна поддерживать управление моментальными снимками виртуальных машин.
20Платформа должна поддерживать управление моментальными снимками виртуальных дисков.
21Платформа должна поддерживать функционал автоматической балансировки/оптимизации нагрузки между вычислительными узлами (Worker Nodes).
22Платформа должна поддерживать обновление платформы без остановки и перебоев в работе платформы.
23Платформа должна иметь API для управления и интеграции с другими системами.
24Облачная платформа в части восстановления после катастроф (Disaster Recovery, DR) должна обеспечивать конфигурирование процесса восстановления после катастроф для инстансов и данных для сервисов на платформе, в том числе для реляционных и нереляционных баз данных.
25Облачная платформа в части восстановления после катастроф (Disaster Recovery, DR) должна обеспечивать конфигурирование процесса восстановления после катастроф для IaaS (виртуальные машины, Microsoft Windows, Linux, блочное хранилище, программно-определяемое хранилище и т.д.).
26Облачная платформа в части восстановления после катастроф (Disaster Recovery, DR) должна обеспечивать восстановление после катастроф в резервном ЦОД для подходов Warm Standby и Multi-site active/active.
