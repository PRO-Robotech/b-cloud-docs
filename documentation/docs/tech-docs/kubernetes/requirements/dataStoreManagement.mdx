# 2.4. Данные

## 2.4.1 Требования к системе управления хранением данных

<table>
  <thead>
    <tr>
      <th>№</th>
      <th>Требование</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>1</td>
      <td>Платформа должна поддерживать возможность использования СХД с блочным доступом к данным</td>
    </tr>
    <tr>
      <td>2</td>
      <td>Платформа должна поддерживать возможность использования СХД, подключенных по протоколу FC</td>
    </tr>
    <tr>
      <td>3</td>
      <td>Платформа должна поддерживать возможность использования СХД, подключенных по протоколу iSCSI</td>
    </tr>
    <tr>
      <td>4</td>
      <td>Платформа должна поддерживать возможность использования СХД, подключенных по протоколу NFS</td>
    </tr>
    <tr>
      <td>5</td>
      <td>Платформа должна поддерживать возможность использования СХД, подключенных по протоколу CIFS/SMB</td>
    </tr>
    <tr>
      <td>6</td>
      <td>Платформа должна поддерживать подключение к СХД без использования монопольного доступа к оборудованию</td>
    </tr>
    <tr>
      <td>7</td>
      <td>Платформа должна поддерживать прямой проброс томов в ВМ по протоколу FC</td>
    </tr>
    <tr>
      <td>8</td>
      <td>Платформа должна поддерживать прямой проброс томов в ВМ по протоколу iSCSI</td>
    </tr>
    <tr>
      <td>9</td>
      <td>Платформа должна поддерживать возможность организации распределенного хранения данных между удаленными площадками на базе СХД с блочным доступом к данным (только для Master Nodes)</td>
    </tr>
    <tr>
      <td>10</td>
      <td>Платформа должна поддерживать вертикальное и горизонтальное масштабирование СХД без прерывания работы платформы для компонентов платформы</td>
    </tr>
    <tr>
      <td>11</td>
      <td>Платформа должна поддерживать вертикальное и горизонтальное масштабирование СХД без прерывания работы платформы для сервисов IaaS/PaaS/SaaS</td>
    </tr>
    <tr>
      <td>12</td>
      <td>СХД в составе Платформы должна поддерживать технологии шифрования данных</td>
    </tr>
    <tr>
      <td>13</td>
      <td>Платформа должна поддерживать динамическое выделение для дисков ВМ (технология Thin Provisioning)</td>
    </tr>
    <tr>
      <td>14</td>
      <td>Платформа должна поддерживать отказоустойчивое подключение к системе хранения данных (MPIO) для FC, iSCSI, S3</td>
    </tr>
    <tr>
      <td>15</td>
      <td>Платформа должна поддерживать управление S3: увеличение размера бакета, создание бакетов, резервное копирование</td>
    </tr>
  </tbody>
</table>
