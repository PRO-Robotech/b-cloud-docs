# 2.9. Логирование
## 2.9.1. Базовые требования к логированию

<table>
  <thead>
    <tr>
      <th>№</th>
      <th>Требование</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>1</td>
      <td>Платформа должна обеспечивать регистрацию событий по всем компонентам облачной инфраструктуры (дата и время события, идентификатор, объект, результат события), включая изменения конфигурации компонентов облачной платформы, а также ВМ и сервисов, развернутых на платформе</td>
    </tr>
    <tr>
      <td>2</td>
      <td>Платформа должна поддерживать механизмы разделения событий по уровню критичности (Critical, Warning, Informational)</td>
    </tr>
    <tr>
      <td>3</td>
      <td>Платформа должна поддерживать механизмы защиты журналов от изменений</td>
    </tr>
    <tr>
      <td>4</td>
      <td>Платформа должна поддерживать механизмы настройки уровня логгирования для каждого компонента облачной платформы</td>
    </tr>
    <tr>
      <td>5</td>
      <td>Платформа должна поддерживать сквозной поиск событий по всем компонентам облачной платформы</td>
    </tr>
    <tr>
      <td>6</td>
      <td>Платформа должна поддерживать фильтрацию отображения журнала по типам событий, объектам, уровням критичности</td>
    </tr>
    <tr>
      <td>7</td>
      <td>Платформа должна поддерживать сбор и централизованное хранение данных, событий, метрик о состоянии программных и аппаратных компонентов облачной платформы</td>
    </tr>
  </tbody>
</table>
