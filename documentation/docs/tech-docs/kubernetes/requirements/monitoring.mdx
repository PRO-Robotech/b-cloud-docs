# 2.8. Мониторинг
## 2.8.1. Базовые требования к мониторингу


<table>
  <thead>
    <tr>
      <th>№</th>
      <th>Требование</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>1</td>
      <td>Платформа должна обеспечивать возможность мониторинга доступности, сбора и отображения данных о состоянии всех компонентов/сервисов управления.</td>
    </tr>
    <tr>
      <td>2</td>
      <td>Платформа должна обеспечивать возможность проактивного и событийного мониторинга инфраструктуры облачной платформы, определения узких мест и потенциальных проблем.</td>
    </tr>
    <tr>
      <td>3</td>
      <td>Платформа должна обеспечивать возможность сбора данных и метрик по утилизации компонентов облачной платформы.</td>
    </tr>
    <tr>
      <td>4</td>
      <td>Платформа должна поддерживать сбор метрик в разрезе компонентов платформы, ВМ и сервисов, развернутых на платформе.</td>
    </tr>
    <tr>
      <td>5</td>
      <td>Платформа должна поддерживать сбор метрик утилизации процессора компонентов вычислительных ресурсов.</td>
    </tr>
    <tr>
      <td>6</td>
      <td>Платформа должна поддерживать сбор метрик утилизации оперативной памяти компонентов вычислительных ресурсов.</td>
    </tr>
    <tr>
      <td>7</td>
      <td>Платформа должна поддерживать сбор метрик производительности и утилизации сетевых интерфейсов компонентов вычислительных ресурсов.</td>
    </tr>
    <tr>
      <td>8</td>
      <td>Платформа должна поддерживать сбор метрик утилизации дисковой подсистемы компонентов вычислительных ресурсов по объему.</td>
    </tr>
    <tr>
      <td>9</td>
      <td>Платформа должна поддерживать сбор метрик утилизации процессора ВМ (сервисов), развернутых на платформе.</td>
    </tr>
    <tr>
      <td>10</td>
      <td>Платформа должна поддерживать сбор метрик утилизации оперативной памяти ВМ (сервисов), развернутых на платформе по объему.</td>
    </tr>
    <tr>
      <td>11</td>
      <td>Платформа должна поддерживать сбор метрик производительности и утилизации сетевых интерфейсов ВМ (сервисов), развернутых на платформе.</td>
    </tr>
    <tr>
      <td>12</td>
      <td>Платформа должна поддерживать сбор метрик утилизации дисковой подсистемы ВМ (сервисов), развернутых на платформе.</td>
    </tr>
    <tr>
      <td>13</td>
      <td>Платформа должна поддерживать возможность отображения собираемых данных и метрик на порталах самообслуживания и администрирования.</td>
    </tr>
    <tr>
      <td>14</td>
      <td>Платформа должна поддерживать возможность управления и отображения параметров мониторинга через графический интерфейс (GUI).</td>
    </tr>
    <tr>
      <td>15</td>
      <td>Необходима интеграция с банковскими системами мониторинга.</td>
    </tr>
    <tr>
      <td>16</td>
      <td>Обеспечение предоставления алертинга по заданным параметрам.</td>
    </tr>
    <tr>
      <td>17</td>
      <td>Платформа должна поддерживать возможность настройки оповещения по событиям, триггерам, статусам путем отправки почтовых сообщений.</td>
    </tr>
    <tr>
      <td>18</td>
      <td>Платформа должна позволять настраивать защиту от переполнения данных, в т.ч. проводить очистку данных.</td>
    </tr>
  </tbody>
</table>
